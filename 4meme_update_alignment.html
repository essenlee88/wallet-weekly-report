4meme 更新对齐 — OpenFour

OpenFour 模块化发射引擎 · 三种 Preset · 来源：技术文档（third-party-integration-0514.md）+ 与 Adam（4meme）群聊确认 · 2026-05-16
一句话能懂版

旧 4meme：一套合约走天下。发 token 规则固定：内盘 bonding curve，毕业全去 PancakeSwap，没有税，没有差异。

OpenFour 做了什么：

  1. 把"合约"变成可拼装的模块 — 每个 token 由 6 个模块组合而成（token 本体、金库、定价曲线、交易规则、迁移目标、扩展数据）。不同组合 = 不同 Preset（类型）。
  2. 三种玩法（Preset） — 创作者激励（外盘税归创作者）、Likwid 毕业（去新 DEX）、金库（外盘税进链上金库）。
  3. Dev 不能改合约 — 选了 Preset 就锁死，项目方无法在发行后修改规则，安全边界清晰。
  4. 识别方式变了 — 旧版靠合约地址特征，新版靠 encodedTags（创建事件里的 57 字节快照），可离线判断 token 类型，不需要额外链上查询。
OpenFour 框架说明

把合约做成可拼装的积木

旧版 4meme 所有 token 同一套规则。OpenFour 把合约拆成了几个功能部件（代币本体、资金保管、定价方式、交易规则、毕业去向），每个 token 创建时把这些部件组合好，之后不可修改。

不同的组合方式就是不同的「Preset（玩法类型）」。4meme 目前提供三种固定 Preset（见下方），dev 只能选其一，不能自定义合约行为。

识别方式：每个 token 创建时链上会附上一个「印章」，可以直接判断是哪种 Preset，不需要额外查询链上。

Dev 正在评估 OpenFour 协议接入工作量，以下事项待结论：

合约接入工作量

Likwid DEX 路由支持需在安全评估通过后再排期，当前为 Blocked 状态。

三个 Concern & 决策
1
Likwid DEX 风险评估

Likwid 是 4meme 的新战略合作 DEX。Likwid 毕业 Preset 的 token 毕业后流动性流向这里，而非 PancakeSwap V2。未知 DEX 存在合约漏洞、流动性操控、异常路由等风险。

待处理获取 Likwid Router 合约地址（module.migrate.bonding_likwid tag 对应合约）后，安全侧评估 DEX 合约风险，确认路由逻辑是否存在不可预期资金扣留，再决定是否接入交易路由。

2
Icon 展示方案 & 筛选策略

OpenFour 三种 Preset 需要在 MemeRush 前端做视觉标识，同时平衡"流量不被拆分"的社区感知。

已决策加新样式 icon，不新增 Preset 筛选项。OpenFour 整体作为一个协议进筛选，Preset 内部差异仅通过 icon 标识。

待与 4meme 确认4meme 官方前端是否对三种 Preset 有 UI 样式区分？有的话 icon 需对应适配。

3
自定义合约安全评估要点

当前阶段（三种固定 Preset）：Dev 无法修改已部署合约，安全边界清晰。风险来自 4meme 协议层（模板升级权），不来自个别项目方。

未来阶段（dev 自定义合约）：攻击面显著扩大——税率隐藏、蜜罐、貔貅机制均可能通过自定义合约或 Hook 层注入。

4meme 审计方案（已确认）：自定义合约提交后经双层审计——外部 Salus + 内部 Binance Security，通过后才能上线。

待处理在 4meme 开放自定义合约（预计上线后 1–2 周），需：① 提交 SDL，明确平台侧风险边界；② 与 4meme 确认自定义标签的数据字段和内容格式；③ 验证数据侧和交易侧能兼容自定义合约结构。
