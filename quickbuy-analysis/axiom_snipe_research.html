Axiom Migration Snipe 机制调研

Task #130 关联 · 2026-04-23 · 基于实操截图的竞品分析

一、核心发现

Axiom 在 Token 接近毕业（Migrating）时，不走 Bonding Curve 内盘买入，而是提供专用的 Migration Snipe 功能——用户提前设置参数并给出 Bribe Fee（Jito Tip），由服务端在迁移完成的瞬间通过 Jito Bundle 抢入外盘第一批交易。

如果用户提前布置了 Sniper 并给出足够高的 Bribe Fee，在 Migrating 窗口期确实有结构性优势。

二、机制对比：普通买入 vs Axiom Snipe

普通 Buy（GMGN / Binance） Axiom Migration Snipe
Migrating 期间行为 路由到 Bonding Curve 内盘直接成交 不走内盘，预挂单等外盘开盘
触发方式 用户点击 → 客户端立即构建+广播 用户预挂 → 迁移完成后自动触发交易（具体机制待确认）
执行机制 用户浏览器（客户端） 两种可能（见下方分析）：
① 链上限价单（合约自动执行）
② 服务端监听 migration 后发市价单
VPN / 用户网络影响 全链路影响（发现+构建+广播） 仅影响预挂环节；无论哪种机制，交易执行本身不经过用户网络
Fee 结构 仅 Priority Fee Priority Fee + Bribe Fee（Jito Tip），平台建议 Bribe ≥ 0.044 SOL (~$3.7)

Snipe 执行机制：两种可能

可能 A：链上限价单（合约自动执行）
用户提交 Snipe → 链上部署一笔条件触发的限价单 → DEX 池子上线时合约自动 match
  • 交易由链上合约触发，不需要服务端监听
  • Bribe Fee 作为 Jito Tip 打包进交易，确保优先上链
  • 优势：去中心化、不依赖服务端可用性
  • 劣势：链上 Gas 开销更高、合约逻辑复杂
可能 B：服务端监听 migration 后发市价单
用户提交 Snipe → 参数存储在 Axiom 服务端 → 服务端监听链上迁移事件 → 触发后立即构建 Jito Bundle 广播
  • 交易由服务端触发，类似中心化 Sniper Bot
  • Bribe Fee 作为 Jito Tip 确保在 migration block 中优先打包
  • 优势：执行速度更快（服务端直连 RPC/Jito Relay）
  • 劣势：依赖平台服务端可用性和诚信
无论哪种机制，核心结论一致：

用户网络（VPN 延迟）不在交易执行的关键路径上。
用户只需提前完成预挂单，实际交易由链上合约或服务端执行。

确认方法：抓包分析 Snipe 提交后的网络请求，或链上查找 Axiom 相关的限价单合约。

四、Axiom Snipe 用户流程还原

Step 1 用户点击 "Snipe RED BULL" 或进入 Adv. → Migration 高级订单
→ 配置金额 + 滑点 + Priority Fee + Bribe Fee

Step 2 提交订单，平台校验参数（Bribe ≥ 0.000001 SOL），失败则返回错误

Step 3 进入等待状态，页面显示迁移进度（9% → 23% ...）
  客户端持续轮询：ping 保活 · gethealth 检查 RPC · getTipAccounts 预取 Jito Tip 地址 · aura 同步状态

Step 4 迁移完成 → 交易自动执行（通过 Jito Bundle 优先打包）

Step 4 的执行方是链上合约还是 Axiom 服务端？目前从截图无法确认，需进一步抓包或链上合约分析。

六、对我们的启示

这不是速度优化能追平的差距，是功能层缺失
Axiom 的 Migration Snipe 是一个独立的产品功能，不是简单的交易加速
  • 当前状态：我方和 GMGN 在 Migrating 期间走 Bonding Curve 内盘买入，用户操作时效性完全依赖客户端网络链路
  • Axiom 优势：用户提前预挂 + 高 Bribe Fee → 迁移完成瞬间自动通过 Jito Bundle 抢入（具体是链上限价单还是服务端市价单待确认），用户网络延迟不影响成交
  • 适用人群：愿意为抢开盘支付额外费用（~$3.7 Jito Tip）的高频/专业交易者
  • 待确认：实际成功率和价格优势有多大（需链上数据验证 Axiom 在 migration block 的占比）

附录：内盘末期 vs 外盘首买 —— 哪个价格更优？

Axiom 选择放弃内盘、等外盘 Snipe，是否真的在价格上有优势？这需要链上数据验证。

Bonding Curve 毕业的三个阶段与价格变化

阶段 1：接近毕业（Bonding Curve ~85-99%）
→ 内盘可交易，价格沿曲线上升，越接近毕业价格越高

阶段 2：Dead Zone（Curve 100%，迁移进行中）
→ 内盘锁定，外盘未创建，无法交易（数秒~数十秒）

阶段 3：外盘上线（DEX 池子创建）
→ 初始价格由迁移注入的流动性决定，通常 ≈ 内盘毕业价
→ 但首批买入推高价格，越早买价格越低

  • 如果外盘首买通常更贵 → Snipe 价值主要在"抢到"而非"便宜"，优先级可调
  • 如果差异很小 → 直接内盘买就够了，Snipe 是锦上添花
