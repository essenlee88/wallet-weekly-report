必答 / Three Core Questions

🎯 三个核心问题

Q01 · 解决哪个业务问题
BSC Meme 做空机制缺失：价格失真、用户无处对冲、平台与牛市共存亡
市场层面：无做空 = 大户拉盘无成本（价格发现失灵）；下跌时没有空头回补做缓冲，流动性死亡螺旋加速；Rug 出货无外部制衡；利润只来自"比别人先跑"，链上分析价值归零，机器人横行。

用户层面：链上洞察无法货币化（预测到崩但没有工具变现）；持多头无法 Delta 对冲；熊市行情用户完全无事可做——平台交易量与大盘相关性 100%，抗周期能力为零。

市场验证：alt.fun（HyperEVM 杠杆 meme 平台）上线 12 天吸引 6,258+ tokens、累计 $30M+ 交易量。BSC 链上同类市场目前空白。
Q02 · 改变哪个用户行为
从「只能做现货」变成「可以直接做多 / 做空」
现在：买入后只能等涨，跌了只能割肉离场，下行行情完全无解
↓ 之后：发现已毕业 Meme Token → 若有 Vault 覆盖，L↑ 做多 / S↓ 做空一键开仓 → My Positions 加减仓 / 平仓
下跌行情中同样可以获利，不再只是「先跑赢」
覆盖范围：已从 Bonding Curve 毕业（Migrate）且 Likwid 有活跃 Vault 的代币子集
Q03 · 30 天如何验证
部署原型，看真实链上开仓数量
前端原型接 Likwid BSC Mainnet 合约上线，邀请目标用户测试

02 / User Insight

👥 目标用户 & 当前阻碍

目标用户
  • 有杠杆需求的活跃 Meme 交易者：习惯用合约思维操作（Margin Level / PnL / 加减仓），现货已满足不了风险偏好和资金效率
  • 想做空 Meme 的用户：链上有洞察（识别 Rug、识别拉盘见顶），但 BSC 上没有任何工具能把这种判断变现
→
当前阻碍
  • MemeRush 只有现货——用户发现已毕业 Meme Token 后，即使想加杠杆或做空，也必须跳出平台
  • 下跌行情中用户零工具可用，只能离场；平台黏性完全依赖牛市
  • 链上分析洞察无法变现：预测到崩但无做空路径，信息优势归零
  • 竞品（Bullx / GMGN / Dexscreener）同样无此能力；BSC 链上 Meme 杠杆市场空白（alt.fun 在 HyperEVM，不覆盖 BSC）
03 / Core Insight

💡 核心洞察

洞察
用一个功能修复 BSC Meme 市场的结构性缺陷，同时让 MemeRush 第一次对周期不敏感——从牛市专用工具，变成全天候主战场。
Likwid 是部署在 BSC 的去中心化 AMM 协议，已上线 LikwidVault + LikwidMarginPosition 合约，支持任何已有 Vault 的 Token 做最高 5× 隔离保证金杠杆交易。MemeRush 毕业代币需单独接入 Likwid Vault 才能支持杠杆交易，初期覆盖范围有限；验证阶段以少量代币起步，证明需求后再扩大覆盖。

更关键的是：合约（margin trading）是圈内用户的母语。Margin Level、PnL、加减仓、清算价是他们每天在 Binance 合约上做的事，学习成本为零。我们做的不是引入新机制，而是第一次给 BSC Meme 用户提供「做空 / 对冲」的能力。

结论：链上合约层无需自建——Likwid 合约已部署、已审计，不需要写一行 Solidity。前端集成复杂度取决于 MemeRush 现有 LiquidMesh 交易框架的兼容性，需要评估交易兼容方案后才能确认工期（详见风险 #1）。

⚙️

合约层：零开发

直接调用 Likwid 已部署、已审计合约。hackathon 范围内不写一行合约代码。

🔗

Vault 覆盖范围有限，需主动部署

Token 毕业到 BSC ≠ 自动支持多空。Likwid Pool 是独立协议，需额外部署（createPool）+ LP 注入流动性，与毕业流程无关、不自动创建。初期覆盖少量代币，验证需求后逐步扩大。

👛

钱包即账户

用户用 MemeRush 已连接的 BSC 钱包直接交互，零注册、零 KYC、零跨链摩擦。

05 / Why Binance Wallet

🏆 为什么 Binance Wallet 适合做这件事

🌐

BSC 原生优势

Likwid 部署在 BSC，Binance Wallet 是 BSC 最大钱包入口。流量和合约在同一条链，天然低摩擦。

📊

发现层已有，只差交易层

MemeRush 已是活跃 Meme 交易者的 Token 发现工具。补上多空交易层，就是一个完整的「发现 → 多空 → 管仓」闭环。

🔧

市场修复者，而非先到先得

BSC Meme 做空缺失需要流量 + 发现层 + 用户聚集三者同时具备才能形成多空生态的 network effect。Bullx / GMGN 缺流量入口，纯 DEX 缺发现层；Binance Wallet × MemeRush 是唯一同时具备的载体。

30 天内只有少量 Migrated token 有活跃 Vault（预计 10–20 个），用户选择受限 		中 先上线验证用户行为，覆盖率是 Phase 2 与 Likwid 深度合作后可扩展的问题，不影响核心验证目标
3 Likwid 流动性深度不足
Vault TVL 较小时，大仓位开仓滑点高；协议处于早期 		低 验证阶段聚焦小仓位行为验证（50 钱包开仓），不依赖大 TVL；MemeRush 导入流量本身有助于 TVL 成长

10 / Long-term Vision

🔭 MemeFlip 作为独立产品的终点

核心定位
BSC meme 的链上期货市场——不是给 MemeRush 加个按钮，而是一个用户专程来做方向性交易的独立产品。
参照系：alt.fun 在 HyperEVM 上做的事，MemeFlip 在 BSC 做——但多一层社交 + 信息维度，让交易不只是杠杆，而是有 alpha 可挖的博弈市场。
🔍

发现层

已毕业 BSC token 列表 + 每个 token 的多空未平仓量 (OI) 比例 + 隐含资金费率。这两个数字在 meme token 维度上全球没有任何平台有——是 MemeFlip 的数据护城河。

⚡

交易层

Likwid 合约支撑的 1–5x 杠杆多空。Phase 2 接入 keeper 实现止损止盈自动触发；实时仓位健康度监控，清算价可视化。

👥

社交层

链上仓位公开可见；KOL 持仓跟随（一键 copy trade）；Short Leaderboard 实时展示哪些 token 被最多聪明钱做空。

为什么社交层是护城河，而不是加分项
  • 在 GMX / dYdX，你只知道价格，因为 BTC 走势只看宏观。
  • 在 MemeFlip，「某大户正在做空 WAIFU」本身就是 alpha——可能是内幕，可能是轧空机会。
  • 链上仓位数据 × meme 社区文化 = 一个专用博弈场，GMGN / Bullx 无法复制，因为他们没有交易层，也没有 Binance 流量做 Vault 冷启动。
最大约束：Vault 流动性冷启动
Token 有 Vault 才能开仓，而 Vault 需要初始 LP。短期解法：ops 为 Top 10 高交易量已毕业 token 主动 seed 初始流动性；一旦有真实交易量，协议手续费自然吸引外部 LP 进入。这是运营问题，不是技术问题。
一句话 Pitch

BSC Meme 市场缺失做空机制已是结构性失灵——MemeRush 用 3 周工程量接上 Likwid 合约，零合约开发就能让用户在发现机会的地方直接做多、做空、对冲，把平台从牛市专用工具升级为全天候主战场。
